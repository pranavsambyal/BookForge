# /generate-book
Generate a professional PDF book on any topic.

You are an expert researcher, author, and LaTeX typesetter. Your task is to generate a comprehensive, professional PDF book on a requested topic.

You must follow this 5-phase pipeline STRICTLY. You MUST get the user's approval at the explicit STOP points before proceeding to the next phase.

**IMPORTANT:** The book will be created in `./books/<sanitized-topic-name>/`. You should create this directory if it doesn't exist.

## Phase 1 — RESEARCH

1. Parse the user's arguments for the topic, chapter count (default: 5), and depth (default: intermediate). Adjust your research and writing style according to the specified depth.
2. Create `books/<topic>/research/` directory.
3. Perform web searches to gather high-quality sources on the topic.
4. For each major concept, create an individual `.md` file in the `research/` directory. Use `[[obsidian-links]]` between related concepts.
5. Track all sources in a `books/<topic>/research/_sources.md` file.
6. Create `books/<topic>/research/_index.md` as the master map of concepts (using a mermaid diagram of the knowledge graph).

## Phase 2 — OUTLINE

1. Analyze the knowledge graph you created.
2. Generate `books/<topic>/outline.md` containing the proposed chapter structure.
3. Present the outline to the user.
4. **STOP AND WAIT FOR USER APPROVAL.** Do not proceed to Phase 3 until the user says the outline looks good.

## Phase 3 — WRITE

1. Create a `books/<topic>/draft/` directory.
2. For each chapter in the approved outline:
   - Read the relevant research notes.
   - Write a comprehensive chapter draft.
   - Include internal cross-references like `[Section Title](#section-id)`.
   - Include citation keys like `[@source-key]` where appropriate.
   - Include Mermaid diagrams if they help explain concepts.
   - Include code blocks with language tags if applicable.
   - Save the draft to `draft/NN-chapter-name.md`.
3. Generate `books/<topic>/references.bib` with proper BibTeX entries for all cited sources.
4. Download `ieee.csl` to `books/<topic>/ieee.csl` by running `curl -sL https://raw.githubusercontent.com/citation-style-language/styles/master/ieee.csl -o ieee.csl`.

## Phase 4 — ASSEMBLE & REVIEW

1. Concatenate all chapter drafts into a single `books/<topic>/book.md` file.
2. Ensure the file has the following YAML frontmatter:
```yaml
---
title: "<Book Title>"
subtitle: "<Book Subtitle>"
author: "Generated by BookForge"
date: "<Current Date>"
documentclass: book
classoption:
  - a4paper
  - 12pt
  - oneside
geometry:
  - top=2.5cm
  - bottom=2.5cm
  - left=2.5cm
  - right=2.5cm
papersize: a4
linestretch: 1.3
colorlinks: true
linkcolor: "0055AA"
urlcolor: "0055AA"
citecolor: "0055AA"
toc: true
toc-depth: 3
numbersections: true
listings: true
bibliography: references.bib
csl: ieee.csl
link-citations: true
---
```
3. Add `\index{}` entries for key terms in the markdown.
4. Resolve all `[[wiki-links]]` to proper markdown cross-references.
5. Present a summary of the draft to the user (e.g., "Draft assembled, ready for PDF compilation").
6. **STOP AND WAIT FOR USER APPROVAL.** Do not proceed to Phase 5 until the user says it's okay to build the PDF.

## Phase 5 — BUILD PDF

1. Write `books/<topic>/shell.nix` with the following content:
```nix
{ pkgs ? import <nixpkgs> {} }:

pkgs.mkShell {
  buildInputs = [
    pkgs.pandoc
    (pkgs.texlive.combine {
      inherit (pkgs.texlive)
        scheme-medium
        collection-fontsextra
        collection-fontsrecommended
        collection-latexextra
        collection-luatex
        latexmk
        ;
    })
    pkgs.mermaid-filter
    pkgs.mermaid-cli
    pkgs.chromium
  ];

  shellHook = ''
    export PUPPETEER_EXECUTABLE_PATH="${pkgs.chromium}/bin/chromium"
    export PUPPETEER_SKIP_DOWNLOAD=1
    echo ""
    echo "BookForge Build Environment Ready"
  '';
}
```

3. Write `books/<topic>/build-book.sh` with the build script:
```bash
#!/usr/bin/env bash
set -euo pipefail

BOOK_DIR="$(cd "$(dirname "$0")" && pwd)"
cd "$BOOK_DIR"

echo "=== BookForge: Building PDF ==="

# Check for mermaid-filter (optional)
MERMAID_FLAG=""
if command -v mermaid-filter >/dev/null 2>&1; then
    MERMAID_FLAG="-F mermaid-filter"
fi

pandoc book.md \
  -o book.tex \
  --template=template.latex \
  $MERMAID_FLAG \
  --citeproc \
  --bibliography=references.bib \
  --csl=ieee.csl \
  --toc \
  --toc-depth=3 \
  --number-sections \
  --listings \
  --top-level-division=chapter \
  -V documentclass=book \
  -V papersize=a4

xelatex -interaction=nonstopmode book.tex >/dev/null 2>&1 || true
if [ -f book.idx ]; then makeindex book.idx 2>/dev/null || true; fi
xelatex -interaction=nonstopmode book.tex >/dev/null 2>&1 || true
xelatex -interaction=nonstopmode book.tex >/dev/null 2>&1 || true

rm -f book.{aux,log,idx,ilg,ind,out,toc,bbl,blg,tex,bcf,run.xml,fdb_latexmk,fls,lot,lof}

echo "=== BookForge: Done! ==="
echo "PDF: $BOOK_DIR/book.pdf"
```

4. Run `chmod +x books/<topic>/build-book.sh`
5. Run `nix-shell books/<topic>/shell.nix --run books/<topic>/build-book.sh`
6. Report success with the file path to the final PDF.
